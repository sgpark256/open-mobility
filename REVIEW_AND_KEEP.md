# 수정 확인 후 KEEP 하는 방법

## 1. Cursor 안에서 (에이전트·채팅이 파일을 고쳤을 때)

1. 변경된 파일이 열리면 **diff(빨강/초록)** 가 보입니다.
2. 상단 또는 거품 메뉴에서  
   - **Keep** (또는 **Accept**) → 이 파일의 수정을 **에디터에 반영한 상태로 유지**  
   - **Undo** / 거절 → 해당 수정 **취소**
3. 여러 파일이면 파일마다 또는 **Keep All** 로 한 번에 확정할 수 있습니다.  
   - 코드가 여러 파일로 나뉘어 수정된 경우, **반드시 Keep All(전체 유지)** 로 한 번에 받아들이면 누락 없이 반영됩니다.

→ “KEEP”은 Cursor가 제안한 패치를 **최종적으로 받아들인다**는 뜻에 가깝습니다.

---

## 2. 여러 파일이 한꺼번에 수정된 경우

에이전트가 네비게이션·푸터처럼 **여러 HTML을 동시에** 고친 뒤에는, Cursor에서 **Keep All**로 한 번에 수락한 다음 `git diff`로 차이를 확인하는 것이 안전합니다.

---

## 3. Git으로 (나중에도 되돌리고 싶을 때)

이 프로젝트는 **Git**을 켜 두었습니다. 마음에 드는 시점마다 커밋해 두면 **영구 저장(KEEP)** 이 됩니다.

| 하고 싶은 것 | 명령 (터미널에서 프로젝트 폴더 기준) |
|-------------|--------------------------------------|
| 무엇이 바뀌었는지 보기 | `git status` / `git diff` |
| 수정이 마음에 듦 → 저장 | `git add -A` 후 `git commit -m "설명"` |
| 수정이 마음에 안 듦 → 파일 되돌리기 | `git restore 파일경로` 또는 `git restore .` |

**팁:** 큰 수정을 부탁하기 **직전**에 `git commit -am "작업 전 스냅샷"` 처럼 한 번 찍어 두면, 비교·되돌리기가 쉽습니다.

---

## 4. Git 사용자 이름 (선택)

첫 커밋은 이 PC에만 쓰는 로컬 이름으로 했습니다. 본인 이메일로 남기려면 한 번만 설정하세요.

```text
git config user.name "본인 이름"
git config user.email "본인@이메일"
```

(`--global` 을 붙이면 PC 전체 Git에 적용됩니다.)
